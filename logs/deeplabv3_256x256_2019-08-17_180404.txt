help Tiff
 MATLAB Gateway to LibTIFF library routines.
    obj = <strong>Tiff</strong>(filename,mode) creates a <strong>Tiff</strong> object associated with the
    <strong>Tiff</strong> filename in the specified mode:
 
           'r'     open <strong>Tiff</strong> file for reading.
           'w'     open <strong>Tiff</strong> file for writing; discard existing contents.
           'w8'    open <strong>Tiff</strong> file for writing a BigTIFF file; discard 
                   existing contents.
           'a'     open or create <strong>Tiff</strong> file for writing; created files will 
                   be in 32-bit <strong>Tiff</strong> format; any existing file format will
                   be preserved; append image data to end of file
           'r+'    open (do not create) <strong>Tiff</strong> file for reading and writing.
 
    If the mode is not given, the default is 'r'.
 
    When a <strong>Tiff</strong> file is opened for reading, the first image becomes
    current.
 
    If you open a file for writing or appending, <strong>Tiff</strong> automatically creates 
    a default image file directory (IFD)  for writing subsequent data.  
    This IFD has default values as specified in <strong>Tiff</strong> Revision 6.0.
 
    When creating a new image, you must supply values for certain required 
    fields before writing any image data.  Use the setTag method to set 
    these fields, including ImageWidth, ImageLength, BitsPerSample, 
    SamplesPerPixel, Compression, PlanarConfiguration, and Photometric. If 
    the image layout is stripped, you must also specify RowsPerStrip. If 
    the image layout is tiled, you must specify both TileWidth and 
    TileHeight.  Omission of these tags or specifying improper combinations 
    may result in an invalid file. To verify compliance with the <strong>Tiff</strong> 
    specification, use IMFINFO or IMREAD on the newly created file. 
 
    The <strong>Tiff</strong> object provides access to many of the capabilities of the
    LibTIFF library via methods.  In most cases, the syntax of the <strong>Tiff</strong>
    method is similar to the syntax of the corresponding LibTIFF library
    function.
